1. 쿠키
    1) 클라이언트의 상태 정보를 유지하기 위한 기술
    2) 주로 웹 브라우저에 (키, 값) 형태로 저장했다가 다음 요청 시 저장된 쿠키를 함께 전송함
    3) RFC 6265 (쿠키 표준 명시)
        - 최대 300개까지 만들 수 있음
        - 쿠키 하나의 최대 크기는 4096바이트임
        - 하나의 호스트나 도메인에서 최대로 50개까지 만들 수 있음

2. 기본 동작 매커니즘
    1) 클라이언트가 서버에 요청을 보냄
    2) 서버가 쿠키를 생성하여 HTTP 응답 헤더에 실어 클라이언트에 전송함
    3) 클라이언트는 쿠키를 받아 저장함
    4) 클라이언트는 다음번 요청 시 저장해 둔 쿠키를 HTTP 요청 헤더에 실어 보냄
    5) 서버는 쿠키의 정보를 읽어 필요한 작업을 수행함

3. 레이어 팝업
    1) 하루동안 열지 않음
    2) 동작 예시                                  팝업닫기           새로고침(재방문)
        - 메인 화면 (팝업공지 - 하루동안 열지 않음) =======> 메인 화면 ===============> 메인 화면 (팝업공지 - 하루동안 열지 않음)
               
                                                       팝업닫기           새로고침(재방문) 
        - 메인 화면 (팝업공지 - 하루동안 열지 않음(체크)) =======> 메인 화면 ===============> 메인 화면
    
    3) ajax()
        - 비동기 HTTP 요청을 보내는 jQuery 함수임
            - 비동기 : 요청 보내고 응답을 기다리지 않고 다른 행동 하는 것
            - 동기   : 요청 보내고 응답이 올 때까지 기다리는 것
        - 인수
            - HTTP 요청을 구성하는데 사용하는 설정들
            
            - url
                - 요청을 보낼 페이지 url
                - 기본값은 현재 페이지임
            - type
                - get, post 등 http 메서드를 지정
            - data
                - 서버로 보낼 데이터
            - datatype
                - 서버로부터 받을 응답 데이터의 타입
            - success
                - 요청 성공 시 실행할 콜백 함수
