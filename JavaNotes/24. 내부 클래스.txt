1. 내부 클래스(inner class) 의미
    1) 클래스 내부에 선언한 클래스
    2) 이 클래스를 감싸고 있는 외부 클래스와 밀접한 연관이 있는 경우가 많음
    3) 다른 외부 클래스에서 사용할 일이 거의 없는 경우에 내부 클래스로 선언해서 사용
    4) 중첩 클래스라고도 함
    5) 종류 
        - 인스턴스 내부 클래스
        - 정적(static) 내부 클래스
        - 지역(local) 내부 클래스
        - 익명(anonymous) 내부 클래스
    6) 외부 클래스의 내부에서만 사용하려고 내부 클래스를 사용!!!!!!!!
    7) 내부 클래스를 밖에서 호출하거나 하지 않음

2. 인스턴스 내부 클래스
    1) 내부적으로 사용할 클래스를 선언 (private으로 선언하는 것을 권장)
    2) 외부 클래스가 생성된 후 내부 클래스가 생성됨
        OutClass          outClass = new OutClass();
        OutClass.InClass  inClass  = outClass.new InClass(); 
        ----------------             --------
        타입처럼 사용                  객체 안에 생성

3. 정적(static) 내부 클래스
    1) 외부 클래스 생성과 무관하게 사용할 수 있음
    2) 정적 변수, 정적 메서드 사용 가능
    
4. 지역(local) 내부 클래스
    1) 지역 변수와 같이 메서드 내부에서 정의하여 사용하는 클래스
    2) 메서드 호출이 끝나면 메서드에 사용된 지역변수의 유효성은 사라짐
    3) 메서드 호출 이후에도 사용해야 하는 경우 지역 내부 클래스에서 사용하는 메서드의 지역변수나 매개변수는 final로 선언됨

5. 익명(anonymous) 내부 클래스
    1) 이름이 없는 클래스
        - 지역 내부 클래스 이름은 실제로 호출되는 경우가 없음
    2) 클래스의 이름을 생략하고 주로 하나의 인터페이스나 하나의 추상클래스를 구현하여 반환
    3) widget의 이벤트 핸들러에 활용됨
