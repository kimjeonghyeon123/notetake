1. 제네릭(Generic) 자료형
    1) 클래스에서 사용하는 변수의 자료형이 여러개일 수 있고, 메서드는 클래스의 자료형을 특정하지 않고
       추후에 해당 클래스를 사용할 때 지정할 수 있도록 선언
        - 클래스, 메서드에서 사용할 자료형을 나중에 확정하는 기법
        - 객체를 생성할 때나 메서드를 호출할 때 정한다는 의미

    2) 실제 사용하는 자료형의 변환은 컴파일러에 의해 검증되므로 안정적인 프로그래밍 방식

    3) 컬렉션 프레임워크에서 많이 사용되고 있음

2. 제네릭 기반의 클래스 정의
    1) 제네릭 관련 변수 용어
        - 타입 매개변수(type parameter) = 자료형 매개변수 T
            - public class abc<T>{
                 
              }
            - 보통 한 문자로 이름을 지음
            - 대문자로 이름을 지음
                - E(Element)
                - K(Key)
                - N(Number)
                - T(Type)
                - V(Value)

    2) 다이아몬드 연산자 <>
    
    3) <T extends 클래스> 사용하기
        - 상위 클래스 범위를 제한함
        - 상위 클래스에서 선언하거나 정의하는 메서드를 활용할 수 있음
        - T에 무작위 클래스가 못 들어가게 함

3. 제네릭 메서드
    1) 자료형 매개변수를 메서드 매개변수나 반환 값으로 정의
    2) 자료형 매개변수가 하나 이상인 경우도 있음
    3) 제네릭 클래스가 아니어도 내부에 제네릭 메서드는 구현하여 사용할 수 있음

    4) public <자료형 매개변수>반환형 메서드이름(자료형 매개변수)
        - public <T> T showData(T data)
