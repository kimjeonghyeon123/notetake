1. 데이터베이스
    1) 여러 사용자가 공유하여 사용할 수 있도록 통합해서 저장한 운영데이터의 집합
  
    2) 예제
        - 은행 : 계좌번호, 입출금 내역
        - 항공사 : 예약정보, 비행기 스케쥴
        - 대학교 : 학생정보, 수강신청
        - 온라인 쇼핑몰 : 고객기록, 주문내역
        - 제조업 : 제품목록, 주문, 재고, 공급망
        - 회사 : 직원정보, 연봉
    
    3) 특징
        - 실시간 접근 가능
            - (쇼핑몰) 목록 조회
        - 지속적으로 변화
            - (쇼핑몰) 구입 정보, 물건 재고 정보
        - 동시 공유가 가능
            - (쇼핑몰) 많은 고객이 동시 접속, 동시 구매 가능
        - 저장된 주소가 아닌 내용으로 참조 가능
            - (쇼핑몰) 가장 많이 팔린 제품은?
    
    4) 유형
        - 정형 데이터 (structured data)
            - 관계 데이터베이스의 테이블, 엑셀의 스프레드시트
        
        - 반정형 데이터 (semi-structured data)
            - HTML, XML, JSON
        
        - 비정형 데이터 (unstructured data)
            - 정해진 구조가 없이 저장된 데이터
            - txt, 멀티미디어 데이터

2. 파일 시스템을 사용했을 경우의 문제점
    1) 데이터 중복성 : 공간 낭비
    2) 업데이트 및 데이터 일관성(data consistency) 유지에 어려움
    3) 데이터 무결성 유지 어려움
    4) 데이터 종속성
        - 어떤 파일을 보려면 특정 응용프로그램으로만 확인 가능
    5) 동시성 (concurrency) 제공이 어려움
    6) 보안 제공 이슈

3. 데이터베이스 관리 시스템(DBMS : DataBase Management System)
    1) 응용 프로그램과 데이터 파일 분리
    2) 데이터베이스 관리 시스템(DBMS)
        - 응용 프로그램과 데이터를 연결을 도와줌
        - 데이터 관리와 데이터에 대한 기본 처리를 담당하는 추가적인 소프트웨어
        - 데이터베이스에 데이터를 저장하고,
          저장된 데이터를 관리하여 조직에 필요한 정보를 생성해주는 시스템

    3) 파일 시스템의 데이터 중복과 데이터 종속 문제를 해결하기 위해 제시된 소프트웨어
    4) 데이터베이스 생성과 관리를 담당
    5) 모든 응용 프로그램은 데이터베이스 공유 가능, DBMS 통해 데이터 CRUD(삽입, 검색, 수정, 삭제)
    6) DBMS는 OS와 함께 중요한 시스템 소프트웨어 패키지
        - Postgres : 약 80만 코드 라인
        - Oracle   : 약 80만 코드 라인
        - MS SQL Server
        - SAP HANA
        - MySQL
    7) DBMS 주요 기능
        - 데이터베이스 구조를 정의하거나 수행
        - 데이터를 삽입, 삭제, 수정, 검색하는 연산 수행
        - 데이터를 항상 정확하고 안전하게 유지하는 기능

    8) 장점
        - 파일 시스템의 데이터 중복 문제 해결
        - 테이블 만들기
        - 
    
    9) 단점
        - 응용프로그램이 DBMS를 통해 데이터에 접근하기 때문에 추가적인 오버헤드가 있음
        - 응용 프로그래머가 DBMS가 어떻게 동작하는 지와 표준 데이터 언어에 대한 지식 필요
        - DBMS 장애가 발생할 때 모든 응용 프로그램 장애가 발생함

4. 데이터베이스 사용자
    1) 데이터베이스 관리자(DBA)
        - 데이터베이스 시스템을 운영하고 관리
        - 데이터베이스를 설계 및 구축, 제어
        - DBMS 자체는 물론 데이터베이스 구축, 관리에 해박한 지식과 많은 경험이 요구됨

    2) 응용 프로그래머
        - 데이터베이스 언어를 이용하여 응용프로그램을 작성
