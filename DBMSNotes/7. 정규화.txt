1. 정규화(Normalization) 정의
    1) 데이터의 일관성을 유지하고 
       데이터의 중복을 방지하며 
       데이터의 유연성을 유지하기 위해 데이터를 분해하는 과정.
    2) 중복을 제거하기 위한 테이블 정의 규칙

3. 정규화 용어 
    1) 정규화 (Normalization)
        - 함수적 종속성 등과 같은 이론에 근거하여 
          관계형 데이터베이스 테이블의 삽입/삭제/갱신 이상(anomaly) 현상
          발생을 최소화하기 위해 좀 더 작은 단위의 테이블로 설계하는 과정.
        - 정규화는 데이터 모델을 정규형에 맞도록 고치는 과정.  
    2) 정규형 (NF : Normal Form)
        - 정규화한 결과
        - 정규화 결과에 의해 도출된 데이터 모델이 갖춰야 할 특성을 만족한
          행태.
    3) 함수적 종속성 (FD : Functional Dependency) 
        - 테이블의 특정 컬럼 A의 값을 알면 다른 컬럼 B는 컬럼 A에 함수적
          종속성이 있다고 함.
          - 예) 주민등록번호를 알면 고객명을 알수 있을 때
               => 고객명은 주민등록번호에 함수적 종속성이 있다.

    4) 결정자 (Determinant)
        - 함수적 종속성에서 컬럼 A를 결정자라고 함.
        - 예) 주민등록번호는 고객명을 결정하므로 
              주민등록번호는 고객명의 결정자임.

4. 정규화 효과 
    1) 상호 종속성이 강한 데이터 요소들을 분리, 
       독립된 개념(엔터티, 테이블)으로 정의하게 됨 
       => 높은 응집도 & 낮은 결합도 (High Cohesion & loose Coupling)

    2) 정규화를 통해 개념이 좀 더 세분화됨에 따라 
       해당 개념에 대한 재활용 가능성이 높아짐.

    3) 속성이 한 번만 표현됨에 따라 중복이 최소화됨.
       데이터 품질이 확보되고 저장공간이 절약되며 
       DML 처리 시 성능이 향상됨.           

5. 정규화 이론 
    1) 1차 정규화 
        - 한 속성에 여러 개의 속성값을 갖거나 
          같은 유형의 속성이 여러 개인 경우 해당 속성을 분리 시킴.
    2) 2차 정규화
        - 주식별자에 완전 함수 종속되지 않은 속성을 분리함
        - 2차 정규화 작업으로 부분 종속 속성이 된 속성을 분리함 
    3) 3차 정규화
        - 일반 속성끼리 함수 종속이 발생한 속성을 분리함 
        - 3차 정규화 작업으로 이행적 종속 속성을 분리함         

6. 제1정규형 
    1) 한 속성에 여러 개의 속성값을 갖거나 
       같은 유형의 속성이 여러 개인 경우 해당 속성을 분리 시킴야 함.
       - 나눌 수 있을 만큼 쪼개라.

7. 제2정규형 
    1) 제1정규형을 만족하고 엔터티내에서 식별자가 아닌 속성은 
       식별자 전체에 종속되어야 함 
       - 테이블의 컬럼들이 기본키와 직접 연관되는 컬럼만으로 구성하라.
    2) 식별자 전체에 종속적이지 않거나 
       복합 식별자일 경우 식별자들 중 일부 식별자 속성(들)에만 
       종속적인 일반 속성은 분리되어야 함 

8. 제3정규형
    1) 제2정규형을 만족하고 일반 속성들 간에도 함수 종속관계가 
       존재하지 않아야 함
       - 컬럼들 간의 종속관계가 있으면 안됨
    2) 제3정규형을 만족하려면 일반 속성들 간 종속관계가 존재하는 
       것들을 분리되어야 함
