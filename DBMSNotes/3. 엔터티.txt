1. 엔터티 개념
    1) 사람, 사물, 사건, 개념 등의 명사에 해당됨
    2) 비즈니스 관점에서 IT 시스템을 통해 관리가 필요한 관심사에 해당됨
    3) 엔터티와 인스턴스
        - 하나의 엔터티는 여러 개의 인스턴스를 가짐
        - 엔터티는 인스턴스의 집합

2. 엔터티의 표기법
    1) 모서리가 둥근 사각형(바커 표기법 기준)을 그린 후 맨 위에 엔터티명 기재

3. 엔터티 특징
    1) 비즈니스 요구 조건 만족을 위해 반드시 필요하고 저장 및 관리하고자 하는 정보여야 함
        - 상가, 지하철 역
    2) 유일한 식별자에 의해 식별이 가능해야 함
        - 상가번호, 지하철 역 번호
    3) 인스턴스(2개 이상)의 집합이어야 함
        - 상가는 여러 개이다. 지하철 역은 여러 개이다.
    4) 비즈니스 프로세스에 의해 반드시 이용되어야 함
    5) 반드시 속성을 가지고 있어야 함
        - 상가 이름, 지하철 역 이름
    6) 엔터티는 다른 엔터티와 최소 1개 이상의 관계가 있어야 함

4. 엔터티의 분류
    1) 유무형에 따른 분류
        - 유형
            - 실체가 존재, 물리적인 형태가 있음
            - 안정적이고 지속적으로 활용되는 엔터티
            - 예) 고객
        - 개념
            - 물리적인 형태가 존재하지 않음
            - 비즈니스적으로 관리해야 할 개념적 정보를 저장하는 엔터티
            - 예) 보험상품
        - 사건
            - 비즈니스를 수행함으로써 발생되는 엔터티
            - 발생량이 많으며 다양한 통계 자료에 이용될 수 있음
            - 예) 계약

    2) 발생시점에 따른 분류
        - 기본(키)
            - 타 엔터티와의 관계에 의해서 생성되는 것이 아닌 독립적으로 생성이 가능한 엔터티
            - 기본 엔터티는 타 엔터티의 부모 역할을 하게 됨
            - 예) 고객
        - 중심(메인)
            - 기본 엔터티로부터 발생
            - 비즈니스에 있어서 중심적인 역할을 하는 엔터티
            - 타 엔터티와의 관계 속에서 많은 행위 엔터티를 도출시킴
            - 예) 주문
        - 행위(액션)
            - 2개 이상의 부모 엔터티로부터 발생되는 데이터
            - 비즈니스를 처리하는 과정 데이터량이 많아질 수 있음
            - 상세 설계 단계 진행하면서 도출될 수 있음
            - 예) 주문목록

5. 엔터티 네이밍
    1) 가급적 업무 담당자들(현업)이 사용하는 용어를 사용함
    2) 단수명사여야 함
    3) 해당 모델 내에서 유일한 이름이어야 함
    4) 생성 의미에 맞게 지어야 함
