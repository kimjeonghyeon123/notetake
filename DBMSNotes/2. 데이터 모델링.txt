1. 데이터 모델링 
    1) 테이블(데이터)의 근간이 되는 것이 데이터 모델이며
       데이터 모델을 설계하는 과정이 데이터 모델링임.

2. 모델링 정의
    1) 우리가 살고 있는 3차원의 현실 세계를 단순화하여 표현하는 것
                                         -----
                                         서로가 약속한 규약을 표기법이나
                                         언어로 표현    
    2) 현실 세계를 추상화하여 그 구조를 표현한 것 
                  -----
                  일정한 형식에 맞게 표현
    3) 현실 세계에 존재하는 사물이나 사건에 관한 관점 및 연관된 주체
       (사람 혹은 그룹)을 위하여 명확하게 하는 것 
                               ---- 
                               쉽게 애매모호함을 제거하여 표현 

    ==> 복잡한 현실 세계를 추상화, 단순화, 명확화하기 위해 
        일정한 표기법으로 모델을 표현하는 기법      

 3. 데이터 모델링 정의                                 
    1) 현실 세계의 비즈니스를 IT 시스템으로 구현하기 위해 
       데이터 관점으로 업무를 분석하는 기법 
    2) 약속된 표기법으로 표현하는 과정 
    3) IT 시스템의 근간이 되는 데이터베이스를 구축하기 위한
       분석 및 설계의 과정 
    4) 대부분의 IT 시스템은 관계형 데이터베이스(RDBMS)기반으로 
       구축되어 있음
    5) 데이터 모델링을 통해 정의된 데이터 모델을 기반으로 
       물리적인 데이터베이스가 구축되고 SQL문을 활용하여
       데이터가 CRUD(insert, select, update, delete)됨  

4. 데이터 모델링의 3단계

    현실세계 --------------------> 개념적 구조  
              개념 데이터 모델링        |
                                       | 논리 데이터 모델링
                                      \|/
    물리구조(DB)<-------------------- 논리적 구조 
                  물리 데이터 모델링

    1) 개념적 데이터 모델링                  
        - 구현하고 있는 대상에 대해 포괄적인 수준의 데이터 모델링 
        - Conceptual level, Conceptual Schema(개념 스키마)
    2) 논리적 데이터 모델링    
        - 기본키, 속성, 관계, 외래키 
    3) 물리적 데이터 모델링         
        - 논리 데이터 모델을 기반으로 실제 물리 DB 구축 
        - Pysical level, External Schema(외부 스키마)

5. 데이터 모델링의 3요소 
    1) 어떤 것(Thing)
        - 엔터티(Entity)                - 인스턴스(Instance)
        - 예시 : 사원                   - 예시 : 이순신 사원 

    2) 어떤 것의 성격 (Characteristic of a Thing)
        - 속성 (Attribute)                          - 속성값 (Attribute Value)
        - 예시 : 사원은 사원번호, 사원명을 가진다      - 예시 : 사원번호 1번의 사원명은 이순신이다.  

    3) 어떤 것 간의 연관(Association between Thing)        
        - 관계(Relationship)            - 패어링(Pairing)
        - 예시 : 사원은 부서에 속한다     - 예시 : 이순신 사원은 개발팀에 속한다.

6. ERD 그리는 순서 
    1) 비즈니스에 필요한 엔터티를 그리고 속성을 정의함 
    2) 앞서 그린 엔터티를 적절하게 배치
    3) 각 엔터티 간의 관계를 설정
    4) 설정한 관계의 관계명을 기술 
    5) 설정한 관계의 참여도 기술
    6) 설정한 관계의 필수 여부 기술 

        - 데이터 모델링 표기법 
            - Case Method (Barker 표기법)
