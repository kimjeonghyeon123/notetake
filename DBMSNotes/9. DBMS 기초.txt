1. DBMS(DataBase Management System)
    1) 데이터베이스를 관리하는 시스템
    2) 데이터를 저장하고, 유지보수(수정, 삭제, 추가)하고 이를 검색하는 시스템
        - CRUD
    3) 대량의 데이터를 처리한느 시스템
    4) 다양한 자료구조나 검색구조를 사용해 "빠른" 검색 가능
    5) 대부분의 시스템은 R(검색) >>>>> CUD의 빈도수가 많음
    6) 검색에 최적화

2. RDBMS(Relational DataBase Management System)
    1) 관계형 데이터베이스 시스템
    2) 테이블 기반의 DBMS
        - 테이블 : 컬럼형태의 데이터 저장 방식
        - 테이블과 테이블 간의 연관관계(외래키 foreign key)를 이용해 필요한 정보를 구하는 방식
    3) 모델링은 E-R(Entity Relationship) 모델을 사용
    4) 테이블을 엔터티(기본)와 릴레이션십(유도) 테이블로 구분하는 방식
    5) 데이터를 테이블 단위로 관리
        - 하나의 테이블은 여러 개의 컬럼으로 구성됨
    6) 테이블끼리의 중복정보는 최소화시킴
        - 정규화 => 정규형
    7) 여러 테이블을 합쳐 큰 테이블을 생성(JOIN)해서 필요한 정보를 찾아내는 방식

3. 기본용어
    1) 스키마(Schema)
        - DB, 테이블 정의 내역
    2) SQL 쿼리(SQL Query)
        - 관계형 DBMS를 사용하는 전용 질의언어
    3) 기본키(Primary Key : PK)
        - 테이블에서 하나의 레코드를 저장할 수 있는 하나 이상의 컬럼집합
    4) 외래키(Foreign Key : FK)
        - 어떤 테이블의 기본키가 다른 테이블의 컬럼에 들어 있는 경우
    5) 레코드(Record)
        - 테이블에 들어 있는 여러가지 인스턴스 하나하나를 말함
        - PK로 구별 가능

5. 데이터 타입
    1) 문자형
        - CHAR(길이)
          CHARACTER(길이)
            - 고정길이 문자열을 저장함
            - 예) CHAR(5) : 
                4자리 문자열을 저장하면 맨끝에 1자리에 공백으로 채워짐
        - VARCHAR(길이)
          CHARACTER VARYING(길이)
            - (길이 제한이 있는) 가변 길이 문자열을 저장함
            - 예) VARCHAR(5) : 
                컬럼에 4자리 문자열을 저장하면 4자리 문자열만 저장함
        - TEXT
          VARCHAR
            - 길이 제한이 없는 (무제한의) 가변길이 문자열을 저장함
    
    2) 숫자
        - NUMERIC(총자릿수, 소수점자릿수)
            - 예) NUMERIC(7,3)은 최대 9999.999까지 저장 가능
        - NUMERIC(총자릿수)
            - 예) NUMERIC(7)은 최대 9999999까지 저장 가능
        - SMALLINT
            - 정수형
        - INTEGER 혹은 INT
            - 정수형
        - BIGINT
            - 정수형
        - SERIAL
